# Documentation du projet

Organisation actuelle :

- `audits/` : audits fonctionnels et techniques
- `profiles/` : fiches de profils et comparatifs
- `reports/` : rapports de verification et de check-up
- `checklists/` : pages HTML/CSS/JS de recette
- `packaging/` : notes de diffusion desktop, antivirus et builds
- `screenshots/` : captures de reference et de problemes

Le code applicatif ne vit pas ici.
Le code de l'application est dans `src/`.
