RESUME COMPLET DU PROJET ODYSEY
Date de synthese : 2026-04-07
================================


1. IDENTITE DU PROJET
---------------------

Nom actuel :
- Odysey

Nature du projet :
- application de lecture PDF adaptee aux besoins dys
- version desktop Electron
- version web / PWA en preparation avancee

Objectif principal :
- transformer un PDF en version de lecture plus lisible, plus stable et plus adaptable
- reduire la charge cognitive de lecture
- proposer une lecture personnalisee pour differents besoins : dyslexie, dyscalculie, attention, confort visuel, audio

Public vise :
- eleves
- etudiants
- adultes dys
- enseignants
- orthophonistes
- accompagnants

5. PROFILS PRINCIPAUX
---------------------

Profils visibles comme principaux :
- Normal
- Lecture visuelle allegee
- Audio / Focus
- Decodage renforce
- Dyslexie

Logique :
- chaque profil est defini comme un paquet de reglages dans src/profiles.mjs
- l'application applique ensuite ces reglages aux moteurs de lecture, audio, CSS et impression

Calibration recente :
- les profils principaux ont ete simplifes et clarifies
- le comportement des profils a ete stabilise sur plusieurs PDF tests
- les profils structurels ne sont plus surcharges sur les sommaires, auteurs et reperes editoriaux

7. DECODAGE ET SYLLABES
-----------------------

Le moteur de decodage gere :
- coloration pedagogique
- coloration alternative
- sons francais
- syllabification legere ou renforcee
- coupure syllabique configurable
- attenuation des lettres muettes

Le moteur syllabique :
- est concu pour la lisibilite dys
- pas pour une syllabation universitaire parfaite
- utilise des exceptions
- preserve des groupes de sons importants
- est teste par unite

Un dictionnaire local francais a ete integre pour :
- renforcer l'analyse lexicale
- aider l'OCR
- aider a corriger mots colles ou degrades


8. SUPPORT MATHEMATIQUE

18. LIMITES CONNUES
-------------------

Malgre les progres, il reste des limites :
- certains PDF tres riches restent difficiles a reconstruire parfaitement
- certains artefacts de texte peuvent subsister selon la qualite du PDF
- l'OCR peut rester fragile sur des scans tres degradés
- les tableaux complexes et documents tres composites ne sont pas toujours parfaits
- macOS non signe affichera des avertissements
- Linux et macOS doivent etre construits via les bonnes machines ou via CI

22. CONCLUSION
--------------

Odysey n'est plus un simple prototype de lecture PDF.

Le projet dispose maintenant :
- d'une application desktop Windows fonctionnelle
- d'une base de packaging multi-plateforme serieuse
- d'une version web/PWA deployable
- d'un moteur de lecture adapte modulaire
- d'une base OCR locale
- d'une logique de profils documentee
- d'une documentation projet devenue exploitable

Le projet est deja utilisable, testable et diffusable en environnement Windows.
La suite consiste surtout a :
- consolider le comportement sur plus de PDF reels
- calibrer encore les profils
- diffuser proprement la version web
